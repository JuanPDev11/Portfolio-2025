System Overview

The PQR Management System is designed to streamline and automate the request handling process within an organization. The backend, developed with C# and ASP.NET Core 8, uses JWT and OAuth2 for authentication and integrates SignalR for real-time updates across multiple user roles.

The workflow begins with the analyst, who registers the PQR. The request then moves to the coordinator, who forwards it to the approver. Once approved, a response email is sent to the original sender. The system automatically extracts emails and attachments from Outlook when a PQR is created.

During the response phase, the system provides document viewers to display email content and attachments. The response is drafted by the analyst, and a formal letter in PDF format is generated following the company's official template. This letter is then sent back to the original sender.

The backend includes electronic signature management, allowing users to add digital signatures to outgoing emails. It also supports LDAP authentication with Active Directory and an alternative local authentication method using Identity.

Additional functionalities include the CRUD system for managing PQR types, primary and secondary issues, and email senders. The system supports three different workflows:

General Workflow: Analyst → Coordinator → Director (Approval) → Email Response

Executive Workflow: Analyst → Coordinator → HR Director → Audit Director → Final Approval

Simple Workflow: Analyst → Coordinator (Approval) → Email Response

The front-end is built with Angular, implementing Guards, Interceptors, and role-based access management. It ensures a smooth user experience by utilizing real-time updates via SignalR and performance optimizations with Lazy Loading and RxJS.
